# AI 기반 다국어 음성 합성 및 실시간 립싱크 더빙 시스템 개발일지

## 📋 오늘의 작업 내용

### 1. 비동기 작업 큐 필요성

- AI 모델 추론은 CPU/GPU를 점유하며 시간이 오래 걸림 (수 분 ~ 수십 분).
- HTTP 요청은 타임아웃이 있으므로, 요청만 받고 실제 작업은 백그라운드 워커(Worker)에게 위임해야 함.

### 2. 기술 스택 선정

- **Broker**: Redis (가볍고 빠름, 윈도우 지원).
- **Worker**: Celery (Python 표준).
- **Flow**: API -> Redis -> Celery Worker -> AI Module.

## 🔧 기술적 진행사항

### Architecture

- User -> API (Return Task ID) -> (Async) -> Worker -> DB (Update Status).

## 🚧 이슈 사항 및 해결 방안

- **윈도우 호환성**: Redis는 공식적으로 윈도우를 지원하지 않음. -> WSL2를 쓰거나, Memurai(윈도우용 Redis 호환) 사용. 개발 편의를 위해 Memurai Developer Edition 채택.
