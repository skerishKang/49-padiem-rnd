- **날짜**: 2024-10-17
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 1단계 기초연구·설계
- **아젠다**: IRIS 용 내부 데모 시나리오 정의, 콘솔 UI 데모 모드, 립싱/품질 시연용 영상 촬영

---

## 1. 연구 배경·목적 정리 (책임 관점)

- GPU 최적화(10/16) 결과를 바탕으로, 실제 이해관계자(내부·외부)를 대상으로 보여줄 **IRIS 데모 시나리오** 를 구체화하고, 콘솔 UI 데모 모드와 시연용 영상을 준비하는 날.
- 목적은
  - IRIS 과제 목표(립싱/자막/음성 품질 개선)를 직관적으로 보여주고,
  - 품질 수치를 어떻게 시각화해 설득력 있게 제시할지 실험하는 것.

---

## 2. 오늘 진행 내용 (세부)

### 2.1 데모 시나리오 구성
- 교육/마케팅/내부 공지 3개 도메인 시나리오를 선정.
- 각 시나리오에 대해
  - 입력: 원본 음성·자막·영상,
  - 목표: 어떤 언어/스타일로 변환할 것인지,
  - 중점 품질 요소(립싱, 화자 보존, 자막 정합 등)를 명시.

### 2.2 콘솔 UI 데모 모드 개선
- Streamlit 콘솔에 "데모 모드"를 추가하고,
  - 입력 → 처리 → QC 결과 → IRIS 제출 준비 단계까지 한 화면에서 흐름을 볼 수 있도록 구성.
- 품질 지표(립싱 오차, 화자 유사도, 자막 정합도)를 그래프·게이지 형태로 표현하는 방식을 시험.

### 2.3 라이브 시연 영상 촬영
- OBS로 화면(콘솔 UI) + 데모 시나리오 진행 + 결과 영상을 1080p/60fps로 녹화.
- 립싱 오차를 정량적으로 설명하기 위해 Wav2Lip discriminative network 출력을 기반으로 한 그래프 오버레이를 영상에 포함하는 방식을 검토.

### 2.4 UX 피드백 수집
- 내부 UX 리뷰 세션에서 5명으로부터 초기 피드백 수집.
  - 콘솔 용어(메뉴명) 일부가 기술적이라 일반 사용자가 이해하기 어렵다는 의견.
  - 품질 그래프의 범위/색상에 대한 개선 요청.

## 3. 책임자 관점 의사결정·리스크

- **결정 1: 데모 시나리오 공식화**
  - 오늘 정의한 3개 도메인을 IRIS 중간/최종 발표 시 공식 데모 시나리오로 사용하기로 결정.

- **결정 2: 품질 수치의 시각화 방식**
  - 립싱/화자/자막 지표는 숫자뿐 아니라, 기준 대비 상대적 위치(게이지, 색상)를 함께 보여주기로 함.

- **리스크 1: 실시간 데모 안정성**
  - 라이브 시연에서 네트워크/하드웨어 변수가 개입될 위험.
  - → **대응**: 녹화된 데모 영상과 라이브 데모를 병행하여, 장애 발생 시에도 설명이 가능하도록 준비.
