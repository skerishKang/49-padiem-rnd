# 연구노트 (책임자 관점)

- **날짜**: 2024-10-25
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 1단계 기초연구·설계
- **아젠다**: 2차 사용자 테스트(Usability Test), 만족도/품질지표 교차 검증

---

## 1. 연구 배경·목적 정리 (책임 관점)

- 10/22 콘솔 UX 개선과 10/24 IRIS 리허설을 바탕으로, 실제 내부 사용자 그룹을 대상으로 **2차 사용자 테스트**를 진행하는 날.
- 목적은
  - 콘솔 UI와 IRIS 제출 흐름이 사용자 입장에서 자연스럽게 이어지는지,
  - 시스템이 제공하는 품질 지표(립싱/자막/화자 지표)가 사용자 체감 품질과 일치하는지 확인하는 것.

---

## 2. 오늘 진행 내용 (세부)

### 2.1 테스트 그룹 구성
- 연구원 3명, 실제 사용자 역할(강의자/PM 등) 2명으로 총 5명 구성.
- 각자 다른 도메인(교육, 마케팅, 내부 공지)에 해당하는 시나리오를 담당.

### 2.2 시나리오 실행
- 콘솔에서 입력(영상/자막) → AI 처리 → QC 확인 → IRIS 제출 초안 생성까지의 과정을 각자 15분 내에 수행.
- 처리 시간, 오류 메시지, 사용성 이슈를 로깅.

### 2.3 피드백 수집
- 설문(5점 척도) + 반구조화 인터뷰 방식으로 UX와 품질에 대한 의견 수집.
- 주요 피드백:
  - 품질 지표의 설명(툴팁)을 보강해 달라는 요청.
  - IRIS 제출 버튼 위치를 보다 눈에 띄게 배치해 달라는 의견.

### 2.4 품질지표 교차 검증
- 테스트 5건에 대해 QC 리포트를 수집하고,
  - 립싱 오차: 평균 62ms.
  - 화자 유사도: 0.89.
  - 자막 정합률: 99%.
- 주관적 만족도(설문) 평균 4.4/5와 크게 괴리되지 않는 것을 확인.

---

## 3. 책임자 관점 의사결정·리스크

- **결정 1: 지표-체감 매핑 유지**
  - 현재 품질 지표 정의와 사용자가 느끼는 품질이 크게 어긋나지 않으므로, 지표 정의는 유지하되 설명을 강화하는 쪽으로 방향 설정.

- **리스크 1: 권한 정책 이슈 재발 가능성**
  - 일부 시나리오에서 특정 계정이 설정 메뉴에 접근하지 못하는 이슈가 다시 확인됨.
  - → **대응**: 10/30 RBAC 개선 작업에서 해당 계정 유형까지 포함해 매트릭스를 재정비.

---

## 4. 지표·산출물 정리

- **사용성 지표**
  - 사용자 만족도: 평균 4.4/5.
  - 버그 발견: 1건(자막 미리보기 버튼 비활성화) → 즉시 수정.
- **품질 지표**
  - 립싱 오차: 평균 62ms.
  - 화자 유사도: 0.89.
  - 자막 정합률: 99%.
