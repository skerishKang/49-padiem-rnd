# 연구노트 (책임자 관점)

- **날짜**: 2024-11-01
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 2단계 서비스·운영 통합 검증
- **아젠다**: IRIS 리허설용 자동 업로드 스크립트 정비, 무결성 검증, 체크리스트 보완

---

## 1. 연구 배경·목적 (책임 관점)

- IRIS 최종보고 제출을 앞두고, **PowerShell 자동 업로드 스크립트와 IRIS 포털 입력 절차**가 실제 환경에서 안정적으로 동작하는지 리허설.
- 특히, 첨부파일 7종(본문, 요약, 체크리스트, 증빙 ZIP 등)의 **무결성(SHA-256)** 과 IRIS 서버 기록 값이 일치하는지 검증하는 것이 핵심.

---

## 2. 오늘 주요 진행 내용

### 2.1 환경 정비·MFA
- 관리자/검증자 2종 계정에 대해 MFA 설정을 정리하고, 세션 유지 시간을 30분 → 20분으로 단축.
- IRIS 포털 로그인·권한 동작이 자동 스크립트에 영향 주지 않는지 사전 점검.

### 2.2 업로드 스크립트 구조 정리
- `scripts/iris_upload.ps1` 파라미터(과제번호, 출차수, 첨부 경로)를 `.json` 설정 파일로 분리해 재사용성·가독성 개선.
- 리허설/실제 제출을 분리할 수 있도록 모드 플래그(예: `-DryRun`) 설계.

### 2.3 무결성 검증
- 첨부 ZIP 파일의 SHA-256 해시를 계산해 IRIS 서버 응답 JSON과 diff 비교.
- Markdown→HTML 변환 시 줄바꿈 깨짐 문제를 발견하고 `<br/>` 자동 치환 로직 추가.

### 2.4 체크리스트 업데이트
- 리허설 중 발견된 주의사항 6가지를 `최종보고체크리스트.html`에 반영.
  - 예: 파일명 규칙, 첨부 용량 상한, 브라우저 버전, TLS 설정 등.

---

## 3. 책임자 관점 의사결정·리스크

- **결정**: IRIS 제출은 가능하면 자동 스크립트 경로를 기본으로 하고, 수동 입력은 예외 대응용으로만 사용.
- **리스크**:
  - 구버전 Edge(TLS 1.2 강제) 환경에서 스크립트 동작 이슈 → 최소 브라우저 버전 명시 및 사전 안내.
  - 첨부 ZIP 경로 표시 방식에 따라 경고 메시지가 달라질 수 있어, 병기 헤더 추가로 사용자 혼란을 줄이기로 함.
