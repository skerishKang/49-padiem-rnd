# 연구노트 (책임자 관점)

- **날짜**: 2024-11-12
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 2단계 서비스·운영 통합 검증
- **아젠다**: 콘솔 UX 2차 개선, 피드백 라벨링, 만족도 지표 로직 보강, 접근성 재점검

---

## 1. 연구 배경·목적 (책임 관점)

- 그동안 수집된 사용자 피드백(워크숍, 설문, Slido, Teams 등)을 정리해 **콘솔 UX를 한 번 더 손질**하는 날.
- 품질·만족도 지표 계산 로직을 현실 사용 패턴에 맞게 조정하여, 대시보드의 신뢰성을 높이는 것이 또 하나의 목표.

---

## 2. 오늘 주요 진행 내용

### 2.1 피드백 라벨링·분류
- Slido, Teams, 설문 응답 총 68건을
  - issue type
  - severity
  - module(콘솔, IRIS, 교육, 네트워크 등)
  기준으로 라벨링.
- 최상위 요구사항 3가지 도출:
  1) 결과 비교 뷰 고정(좌우 Split-pane, 미니맵 등)
  2) 자동 저장 상태 표시·복구 기능
  3) 접근성(대비·키보드 사용성) 강화.

### 2.2 UX 실험 설계·검증
- 결과 비교 뷰에 대해 3가지 안(Figma 시안)을 만들고, 8명 대상 15분 사용성 테스트 진행.
- SUS 점수 평균 82로, 현행보다 개선된 체감을 확인.

### 2.3 자동 저장·만족도 로직 보강
- Draft 자동 저장 방식을 `localStorage + 서버 백업(버전 5개 롤링)` 조합으로 설계.
- 만족도 지표 계산에서 응답 수가 부족할 경우, 직전 14일 이동 평균으로 대체하도록 로직 수정 → Null 경보 제거.

### 2.4 접근성 개선
- 컬러 팔레트를 WCAG AA 기준에 맞춰 조정(강조 색상·경계선 명도 대비 향상).
- 주요 버튼과 링크에 `aria-label`·키보드 포커스 스타일 보완.

---

## 3. 책임자 관점 의사결정·리스크

- **결정**: UX 개선 항목 중 우선순위 3개(비교 뷰, 자동 저장, 접근성)를 11월 안에 모두 적용, 나머지는 12월 이후로 이관.
- **리스크**:
  - 자동 저장의 서버 백업에 KMS 연동이 필요해, 구현 복잡도가 증가 → 11/15까지 보안팀과 함께 설계 확정.
  - 색상·레이아웃 변경이 기존 사용자에게 낯설게 느껴질 수 있음 → 변경 내역을 릴리스 노트와 교육 자료에 분명히 안내.

---

## 4. 산출물·지표

- UX 개선안 6개 중 3개 우선 적용 결정.
- 만족도 지표 Null 발생률: 12% → **0%**.
- 자동 접근성 테스트(axe-core) 기준 위반 항목: 14개 → 3개로 감소.
