# 연구노트 (책임자 관점)

- **날짜**: 2024-11-13
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 2단계 서비스·운영 통합 검증
- **아젠다**: IRIS 최종 검토 회의, 리스크·의존성 재점검, 승인·의사결정 정리

---

## 1. 연구 배경·목적 (책임 관점)

- IRIS 최종 제출을 앞두고 부서별(연구·재무·품질·보안)로 참여하는 **최종 검토 회의**를 진행.
- 목적은
  - 모든 체크리스트 항목이 충족되었는지 확인하고,
  - 남은 리스크·의존성을 정리한 뒤,
  - 제출·운영 관련 최종 승인·책임 범위를 명확히 하는 것.

---

## 2. 오늘 주요 진행 내용

### 2.1 사전 준비
- 최종 제출 패키지(본문, 요약, 체크리스트, 첨부 ZIP)를 최신 버전으로 묶고 SHA-256 해시 공유.
- 회의 안건을 4개 축으로 구성:
  1) 문서·산출물 검토
  2) 기술·품질 검토
  3) 리스크·이슈 검토
  4) 일정·운영 계획

### 2.2 회의 진행
- 참석자: 연구 4, 재무 2, 품질 2, 보안 1, IRIS 담당 1 (총 10명).
- IRIS 포털 UI 데모, 자동 업로드 로그, CDN 성능 보고서 등을 공유.

### 2.3 리스크·요청 사항 정리
- 리스크 3건 도출:
  1) 자동 저장 암호화(KMS) 안정성 검증 필요
  2) 중국 CDN 장기 모니터링 필요
  3) 교육 미참여 인력 보완 필요
- 각 항목에 대해 책임자/기한을 지정.

### 2.4 승인·의사결정
- IRIS 제출 자체는 **조건부 승인**으로 합의 (잔여 리스크는 제출 전·후 일정 내 처리).
- 최종 책임자는 강혜림(연구), 재무·보안 측의 공동 승인자 지정.
