# 연구노트 (책임자 관점)

- **날짜**: 2024-11-14
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 2단계 서비스·운영 통합 검증
- **아젠다**: IRIS 업로드 스크립트 v1.1 배포, 재무·비용 증빙 패키지 정리, QA 회귀 테스트

---

## 1. 연구 배경·목적 (책임 관점)

- IRIS 최종 제출 직전에 자동 업로드 스크립트를 v1.1로 배포해 안정성과 가시성을 높이고,
- 재무팀 요청에 따라 **집행 내역·비용 증빙**을 하나의 패키지로 정리하는 작업을 수행.

---

## 2. 오늘 주요 진행 내용

### 2.1 업로드 스크립트 v1.1 개선·배포
- PowerShell 모듈화를 강화해 인증·업로드·검증 로직을 함수 단위로 분리.
- 네트워크 실패 시 3회까지 재시도 후 백오프하도록 로직 개선.
- 실행 로그를 JSONL 포맷으로 남기고, Kibana 인덱스 패턴 등록.

### 2.2 재무 증빙 패키지 정리
- 연구비 집행 내역, 인건비, 외주비, 장비 구매·임대 등 증빙을 `docs/finance/2024Q4_IRIS.zip` 으로 묶음.
- 각 항목에 대해 IRIS 양식과 매칭되는 메타데이터(코스트 센터, 과제 코드 등)를 첨부.

### 2.3 QA 회귀 테스트
- 업로드·검증 플로우에 대해 12건의 회귀 테스트 시나리오를 실행.
- v1.0 대비 성공률 92% → 100%로 개선 확인.

---

## 3. 책임자 관점 의사결정·리스크

- **결정**: IRIS 제출은 반드시 스크립트 v1.1을 사용, 수동 제출은 비상시 fallback으로만 허용.
- **리스크**:
  - JSONL 로그 용량 증가(1일 12MB 수준) → 11/20까지 로그 압축·보관 정책 수립 필요.
  - 재무 증빙에 대한 추가 질의 가능성 → FAQ와 함께 재무팀 내 공유 회의 필요.

---

## 4. 산출물·지표

- 회귀 테스트 12건 전부 통과.
- 재무 증빙 ZIP 1개, 내부 승인 3건.
