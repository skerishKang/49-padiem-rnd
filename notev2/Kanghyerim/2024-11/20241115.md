## 1. 연구 배경·목적 (책임 관점)

- 자동 저장 기능에 KMS를 연동해 암호화·키 관리 체계를 정비하고,
- 이전 교육에 참여하지 못한 인력을 대상으로 **보충 교육**을 진행해, 제출 직전 보안·운영 리스크를 낮추는 날.

---

## 2. 오늘 주요 진행 내용

### 2.1 KMS 연동·암호화 구성
- Azure Key Vault에 자동 저장 관련 키를 생성하고, Managed Identity를 사용해 서비스에서 키를 가져오도록 구성.
- 자동 저장된 데이터는 AES-256-GCM으로 암호화, 키 로테이션 주기는 90일로 설정.

### 2.2 암호화 검증
- 무작위로 50건의 자동 저장 데이터를 복호화 테스트.
- 복호화 결과 원본과 일치함을 확인하고, 감사 로그에 key version·요청자 ID를 남기도록 수정.

### 2.3 교육 보충 세션
- 미참여자 3명 + 신규 인력 2명 대상 60분 세션 진행.
- IRIS 제출 플로우·보안 준수사항·자동 저장/KMS 동작 원리를 설명.
- 만족도 4.8/5.

### 2.4 운영 문서 업데이트
- SOP에 KMS 장애 시 복구 시나리오·수동 운영 절차를 추가.
- Teams FAQ 채널에 자동 저장·암호화 관련 Q&A 항목을 보강.

## 3. 책임자 관점 의사결정·리스크

- **결정**: 자동 저장은 항상 암호화 on 상태로 운용, 해제 옵션은 제공하지 않음.
- **리스크**:
  - KMS 요청 쿼터 제한에 걸릴 가능성 → 11/20까지 Azure와 협의해 쿼터 상향 예정.
  - 백업·복구 경로 복잡도 증가 → runbook을 더 구체화할 필요.
