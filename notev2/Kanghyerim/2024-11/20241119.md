# 연구노트 (책임자 관점)

- **날짜**: 2024-11-19
- **작성자**: 강혜림
- **역할**: 책임연구원
- **진행 단계**: 2단계 서비스·운영 통합 검증 (릴리스·UX 배포)
- **아젠다**: 콘솔 UX 개선 배포, 토큰 자동 갱신 옵션 적용, 배포 모니터링, 릴리스 노트 정리

---

## 1. 연구 배경·목적 (책임 관점)

- 11월 초부터 설계·검토해 온 UX 개선(비교 뷰 고정, 자동 저장·상태 표시, 접근성 개선)을 실제 콘솔에 배포.
- IRIS 제출 직전에 로그인 토큰 만료로 인한 중단을 막기 위해 **자동 토큰 갱신 옵션**을 추가.

---

## 2. 오늘 주요 진행 내용

### 2.1 UX 개선 배포
- 프론트엔드 `release-canary` 브랜치에 개선 사항 3개를 합친 후, QA·스테이징을 거쳐 프로덕션에 Canary 배포.
- Cypress E2E 테스트 42건, axe-core 접근성 테스트 18건 수행.

### 2.2 토큰 자동 갱신 옵션 추가
- 업로드 스크립트에 `-AutoRefreshToken` 플래그를 추가.
- 만료 5분 전에 자동으로 갱신 요청을 보내고, 실패 시 사용자에게 명시적인 안내 메시지 출력.

### 2.3 배포 모니터링
- 배포 후 6시간 동안 CPU/GPU/응답 시간/오류율을 집중 모니터링.
- 사용자 세션 128건 중 오류 0건, UX 관련 피드백 2건(긍정) 수집.

### 2.4 릴리스 노트 작성
- `docs/releases/console_v0.9.3.md` 에 개선 항목, 영향 범위, 롤백 방안을 기록.
