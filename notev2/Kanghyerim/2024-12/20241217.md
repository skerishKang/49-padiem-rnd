# 연구노트 (책임자 관점)

- **날짜**: 2024-12-17
- **작성자**: 강혜림
- **역할**: 강혜림
- **진행 단계**: 4단계 운영 안정화·개선
- **아젠다**: 자동화 패치 v1.2 프로덕션 배포, SharePoint 접근성 개선, 롤백 전략 검증

---

## 1. 연구 배경·목적 (책임 관점)

- 리허설을 마친 IRIS 업로드 자동화 패치 v1.2(백오프·관측성 강화)를 프로덕션에 배포하고,
- IRIS 피드백용 증빙 패키지(SharePoint)의 접근성 문제를 해결해 담당자들이 원활히 문서를 열람할 수 있도록 하는 것이 목적.

---

## 2. 오늘 주요 진행 내용

### 2.1 자동화 패치 v1.2 배포
- 10시 기준 Blue-Green 전략으로 v1.1 → v1.2 전환.
- Canary 계정 3개로 업로드 시나리오 실행, Slack 알림·APM·Retry Heatmap이 모두 정상임을 확인.

### 2.2 모니터링
- 배포 후 6시간 동안 Retry Heatmap, Elastic APM, Incident 채널을 집중 모니터링.
- 장애·오류 없이 안정적으로 동작.

### 2.3 SharePoint 접근성 개선
- 산업부 담당자 전용 SSO 게스트 계정을 생성하고, 필요한 문서 폴더(`iris_feedback_bundle_20241216` 등)에만 권한 부여.
- 접근 로그를 감사 폴더에 기록해 누가 언제 무엇을 열람했는지 추적 가능하게 구성.

### 2.4 롤백 전략 검증
- 롤백 스크립트를 QA 환경에서 실행해 v1.1로 8분 이내 복귀 가능한지 테스트.
- 체크리스트에 롤백 경로·조건을 명확히 명시.

---

## 3. 산출물·지표

- 배포 로그 1세트.
- SharePoint 접근 계획 문서 v1.1.
- 롤백 체크리스트 업데이트.

---

## 4. 책임자 관점 의사결정·리스크

- **결정**: IRIS 관련 업로드 자동화는 v1.2를 기본으로 사용하되, 문제가 발생할 경우 롤백 스크립트로 v1.1로 되돌리는 것을 표준 경로로 채택.
- **리스크**: SSO 게스트 계정의 만료·권한 변경 상황을 계속 추적해야 함.
